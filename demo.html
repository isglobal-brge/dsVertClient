dsVert: Privacy-Preserving Vertical GLM, Correlation & PCA

Overview

dsVert enables privacy-preserving statistical analysis on vertically partitioned data across multiple DataSHIELD servers. Each server holds different features for the same observations. No server sees another’s raw data. The analyst (client) receives only aggregate statistics.

All computation uses Ring63 Beaver MPC — pure fixed-point arithmetic with additive secret sharing. No observation-level data is ever disclosed.

Supported analyses

Function Description
ds.psiAlign() Private Set Intersection (EC-DH blind relay)
ds.vertGLM() GLM: Gaussian, Binomial, Poisson (L-BFGS + DCF wide spline)
ds.vertCor() Pearson correlation matrix (Ring63 Beaver cross-products)
ds.vertPCA() Principal Component Analysis (eigen decomposition of correlation)

Setup

library(DSI)
library(DSOpal)
library(dsVertClient)

# Connect to 3 Opal servers (each at a different institution)
builder <- DSI::newDSLoginBuilder()
builder$append(server = "hospital_A", url = "https://opal-a:443",
               user = "analyst", password = "***",
               table = "study.patient_data")
builder$append(server = "hospital_B", url = "https://opal-b:443",
               user = "analyst", password = "***",
               table = "study.patient_data")
builder$append(server = "hospital_C", url = "https://opal-c:443",
               user = "analyst", password = "***",
               table = "study.patient_data")

conns <- DSI::datashield.login(logins = builder$build())
DSI::datashield.assign.table(conns, "D", "study.patient_data")

Step 1: Record Alignment (PSI)

Each server may have different patients. PSI finds the common set without revealing which patients are on which server.

psi <- ds.psiAlign(
  data_name   = "D",
  id_col      = "patient_id",
  newobj       = "D_aligned",
  datasources = conns
)
# PSI result: 158 common observations across 3 servers

Step 2: Binomial GLM (Logistic Regression)

Fit a logistic regression for diabetes prediction. Features are distributed across servers — no server sees the full feature set.

result <- ds.vertGLM(
  data_name = "D_aligned",
  y_var     = "diabetes",
  x_vars    = list(
    hospital_A = c("age", "bmi"),
    hospital_B = c("ped", "glu"),
    hospital_C = c("bp", "skin")
  ),
  y_server  = "hospital_C",
  family    = "binomial",
  datasources = conns
)

Step 4: Correlation Matrix

Compute the full cross-server correlation matrix. Each correlation coefficient is a single aggregate scalar — no individual data disclosed.

cor_result <- ds.vertCor(
  data_name  = "D_aligned",
  variables  = list(
    hospital_A = c("age", "bmi"),
    hospital_B = c("ped", "glu"),
    hospital_C = c("bp", "skin")
  ),
  datasources = conns
)
print(round(cor_result$correlation, 3))

Security Model

Property Guarantee
Observation-level disclosure Zero — only p-dimensional aggregates
Client sees Gradients (sums over n obs), correlation scalars
Servers see Only their own data + random additive shares
Beaver triples Generated server-side (client never sees values)
Transport encryption X25519 + AES-256-GCM between servers
Dealer rotation Different server generates triples each iteration
Collusion threshold Analyst needs (K-1)/K servers for K≥3
