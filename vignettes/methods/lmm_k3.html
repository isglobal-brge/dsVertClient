1. Context

Random-intercept linear mixed model with K = 3 vertical-partitioned covariates. The K = 3 path uses a one-dimensional REML profile over the intra-cluster correlation \(\rho = \sigma_b^2 / (\sigma_b^2 + \sigma^2 / n_i)\): each profile evaluation runs a federated weighted-Gaussian ds.vertGLM with per-cluster GLS weights derived from the candidate \(\rho\), recovering \(\hat\beta(\rho)\) consistently. The variance components \((\hat\sigma_b^2, \hat\sigma^2)\) are then recovered via the Pinheiro & Bates (2000) §2.4.2 within-between ANOVA on the outcome y, with an X-correction step subtracting \(\beta_s'\,\widehat{\operatorname{Var}}_{\text{within}}(X_s)\,\beta_s\) per server to undo the within-cluster X variance absorbed into the raw MSW.

The synthetic longitudinal generator below produces \(n_{\text{clusters}} = 200\) clusters × \(n_{\text{obs}} = 10\) observations = \(n = 2000\) total rows with σ_b² = 0.5, σ² = 1, β = (5, 0.5, 0.3), x_1 ∼ N(0, 1) and x_2 ∼ Bern(0.5) iid within cluster — the well-identified regime for testing the variance- component recovery (Pinheiro–Bates Eq. 2.21).

Non-disclosure invariants

Invariant Statement
D-INV-1 Per-patient covariates never leave their origin server.
D-INV-2 Per-row \(x_{ij}^{\top}\beta + b_i\) never reconstructed.
D-INV-3 Per-patient residuals never revealed; only per-cluster aggregates (cluster sizes + per-cluster sum_y, sum_y²) and per-server pooled within-X cross-products.
D-INV-4 Cluster id and outcome local to label server.
D-INV-5 Cross-server messages signed Ed25519 against dsvert.trusted_peers.

Dataset and partition

2. Data preparation

library(dsVertClient)
library(DSLite)
library(DSI)
library(nlme)
library(ggplot2)
n_clusters <- 200L
n_obs      <- 10L
n          <- n_clusters * n_obs
cluster_id <- rep(sprintf("C%04d", seq_len(n_clusters)), each = n_obs)
patient_id <- sprintf("R%05d", seq_len(n))
b_i        <- rnorm(n_clusters, 0, sqrt(0.5))
b          <- rep(b_i, each = n_obs)
x1         <- rnorm(n)
x2         <- rbinom(n, 1L, 0.5)
y          <- 5 + 0.5 * x1 + 0.3 * x2 + b + rnorm(n, 0, 1)
pooled <- data.frame(
  patient_id = patient_id,
  cluster_id = cluster_id,
  x1 = x1, x2 = x2, y = y)

3. Federated K=3 LMM

fit_ds <- ds.vertLMM.k3(
  y ~ x1 + x2,
  data = "DA", cluster_col = "cluster_id",
  rho_lo = 0.001, rho_hi = 0.999,
  tol = 5e-3, max_outer = 10L,

5. Comparison and verdict

ds_b <- as.numeric(fit_ds$coefficients)
rf_b <- as.numeric(nlme::fixef(fit_ref))
nm   <- names(fit_ds$coefficients)
delta <- data.frame(
  coef   = nm,
  dsVert = ds_b,
  lme    = rf_b)
delta$abs <- abs(delta$dsVert - delta$lme)
delta
#>          coef dsVert    lme      abs
#> 1 (Intercept) 4.9876 4.9928 0.005193
#> 2          x1 0.5162 0.5152 0.001001
#> 3          x2 0.3544 0.3438 0.010611

max_abs    <- max(delta$abs)
sigma_min  <- min(sqrt(diag(as.matrix(vcov(fit_ref)))))
sigma_ratio <- sigma_min / max_abs

sigma_b2_ds  <- as.numeric(fit_ds$sigma_b2)
sigma_b2_ref <- as.numeric(nlme::VarCorr(fit_ref)[1L, 1L])
sigma2_ds    <- as.numeric(fit_ds$sigma2)
sigma2_ref   <- as.numeric(nlme::VarCorr(fit_ref)[2L, 1L])

c(max_abs_delta  = max_abs,
  sigma_min      = sigma_min,
  sigma_ratio    = sigma_ratio,
  sigma_b2_ds    = sigma_b2_ds,
  sigma_b2_ref   = sigma_b2_ref,
  delta_sigma_b2 = abs(sigma_b2_ds - sigma_b2_ref),
  sigma2_ds      = sigma2_ds,
  sigma2_ref     = sigma2_ref,
  delta_sigma2   = abs(sigma2_ds - sigma2_ref))

Federated K=3 REML LMM coefficients vs centralised nlme::lme(REML) fit. Dashed identity line.

The federated K = 3 REML 1-D profile + within-between ANOVA matches the centralised nlme::lme(REML) reference: max|Δβ| = 1.06e-02, σ-ratio 2.3×; \(\hat\sigma_b^2\) = 0.5113 versus reference 0.5048; \(\hat\sigma^2\) = 0.9950 versus reference 1.0002. The X-correction term \(\beta_s'\,\widehat{\operatorname{Var}}_{\text{within}}(X_s)\,\beta_s\) analytically removes the within-cluster X-variance absorbed into the raw MSW estimator (Pinheiro–Bates 2000 §2.4.2), giving \(\hat\sigma^2\) consistent with the centralised REML solution.
