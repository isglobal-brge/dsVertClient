1. Context

Multinomial logistic regression with K classes parametrises the class probabilities via softmax:

\[ \Pr(y = k \mid x) \;=\; \frac{\exp(x^{\top}\beta_k)}{\sum_{\ell=1}^{K} \exp(x^{\top}\beta_\ell)}, \qquad k = 1, \ldots, K, \]

with one reference class fixed at \(\beta_{\text{ref}} = 0\) for identifiability. The negative log-likelihood is convex; Newton updates require the full \((K-1) p \times (K-1) p\) Hessian. The Böhning (1992) Theorem 2 PSD upper bound

\[ H(\beta) \;\preceq\; \tfrac{1}{2}\bigl(I_{K-1} - \mathbf{1}\mathbf{1}^{\top}/K\bigr) \otimes \bigl(X^{\top}X\bigr) \]

replaces \(H\) with a constant majorant — a quadratic surrogate that gives a globally convergent Bohning-MM iteration with no per-iteration matrix inversion in the federated setting.

ds.vertMultinomJointNewton runs the K = 2 OT-Beaver pipeline: the indicator-class encoding lives on the label server, each non-label server contributes its X-block to the Bohning Gram matrix via Beaver vector multiplication, the joint Newton step recovers \((\beta_1, \ldots, \beta_{K-1})\) consistent with nnet::multinom.

Non-disclosure invariants

Invariant Statement
D-INV-1 Per-patient covariates never leave their origin server.
D-INV-2 Per-row \(x_i^{\top}\beta_k\) never reconstructed in plaintext.
D-INV-3 Per-patient class indicator \(\mathbf{1}\{y_i = k\}\) lives only on the label server.
D-INV-4 Bohning Gram updates exchange aggregate Beaver products only.
D-INV-5 Cross-server messages signed Ed25519 against dsvert.trusted_peers.

Dataset and partition

The Anderson iris dataset — 150 flowers × 3 species × 4 numeric measurements — the canonical small multinomial benchmark. We partition the 4 morphometric covariates across 2 servers; the species label lives on s2.

Server Columns Role
s1 patient_id, Sepal_Length, Sepal_Width Sepal block
s2 patient_id, Petal_Length, Petal_Width, species, setosa_ind, versicolor_ind, virginica_ind Petal block + outcome (label)

3. Federated K=2 multinomial joint Newton

The K = 2 Bohning joint Newton runs the Ring127 Chebyshev-exp Beaver round-keyed primitive — which DSLite’s rly expression validator rejects (Syntax error at 'L' on integer-L literals). We therefore use the warm-path multinomial wrapper ds.vertMultinom (same dataset, K = 2 split, OVR + softmax-anchor route) for the in-process demo here. The Bohning joint Newton call signature is identical and runs verde against real Opal / Rserve clusters; that verdict is persisted at scripts/run_opal_demo_probes.R::probe_mnl_joint against the local opal docker cluster.

fit_ds <- ds.vertMultinom(
  species ~ Sepal_Length + Sepal_Width + Petal_Length + Petal_Width,
  data = "DA",
  classes = c("setosa", "versicolor", "virginica"),
  reference = "setosa",

5. Comparison and verdict

ds_B <- as.matrix(fit_ds$beta_po_joint %||% fit_ds$coefficients)
rf_B <- t(as.matrix(coef(fit_ref)))
cn   <- intersect(rownames(ds_B), rownames(rf_B))
cls  <- intersect(colnames(ds_B), colnames(rf_B))
delta <- data.frame(
  coef   = c(t(outer(cn, cls, paste, sep = ":"))),
  dsVert = as.numeric(ds_B[cn, cls, drop = FALSE]),
  multinom = as.numeric(rf_B[cn, cls, drop = FALSE]))
