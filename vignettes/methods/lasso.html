1. Context

The LASSO (Tibshirani, 1996) regularises a Gaussian regression by adding an \(\ell_1\) penalty to the residual sum of squares:

\[ \hat\beta(\lambda) \;=\; \arg\min_{\beta} \frac{1}{2n}\,\bigl\lVert y - X\beta\bigr\rVert_2^2 \;+\; \lambda \,\bigl\lVert\beta\bigr\rVert_1. \]

Beck & Teboulle (2009) show the proximal-gradient (FISTA-accelerated) fixed-point iteration converges linearly:

\[ \beta_{t+1} \;=\; \mathcal{S}_{\lambda / L}\!\bigl( \beta_t - \tfrac{1}{L}\,\nabla\!\bigl(\tfrac{1}{2n}\lVert y - X\beta\rVert_2^2\bigr)\bigr), \]

where \(\mathcal{S}_t(z)\) is the soft-threshold operator and \(L\) is an upper bound on the Hessian spectrum. The gradient depends on the centralised normal-equations \(X^{\top}X / n\) and \(X^{\top}y/n\) which ds.vertGLM(family = "gaussian") already exposes via \(\$\text{covariance}\) and \(\$\text{coefficients}\).

ds.vertLASSOProximal therefore performs no additional MPC rounds beyond the initial federated Gaussian fit: the proximal-gradient iteration runs entirely on the client using only summary statistics the federated fit already returned. At \(\lambda = 0\) the wrapper reproduces the prime fit’s coefficients exactly — a fast handoff sanity check.

Non-disclosure invariants

Invariant Statement
D-INV-1 Per-patient covariates never leave their origin server.
D-INV-2 Per-patient \(x_i^{\top}\beta\) never reconstructed in plaintext.
D-INV-3 The proximal-gradient client iteration consumes only the federated normal-equations summary, not raw data.
D-INV-4 At the non-label servers, partial linear-predictor shares only.
D-INV-5 Cross-server messages signed Ed25519, peer pinned via dsvert.trusted_peers.

Dataset and partition

The Hitters dataset (James, Witten, Hastie & Tibshirani, An Introduction to Statistical Learning, §6.6) — Major League Baseball player career statistics with the response Salary (log-transformed). Complete cases give \(n = 263\), \(p = 19\) covariates. We retain a numeric subset for compactness.

Server Columns Role
s1 patient_id, AtBat, Hits, HmRun, Runs, RBI, Walks Career batting stats
s2 patient_id, Years, CAtBat, CHits, CHmRun, CRuns, CRBI, CWalks Career totals
s3 patient_id, PutOuts, Assists, Errors, LogSalary Defence + outcome (label)

3. Federated proximal-LASSO at OLS limit (\(\lambda = 0\))

fit_gauss <- ds.vertGLM(
  formula = LogSalary ~ AtBat + Hits + HmRun + Runs + RBI + Walks +
                       Years + CAtBat + CHits + CHmRun + CRuns +
                       CRBI + CWalks + PutOuts + Assists + Errors,
  data = "DA", family = "gaussian",
  verbose = FALSE, datasources = conns)

fit_ds <- ds.vertLASSOProximal(
  fit_gauss, lambda = 0,
  max_iter = 2000L, tol = 1e-9,
  keep_intercept = TRUE,
  accelerate = TRUE)

5. Comparison and verdict

delta_wrapper <- max(abs(as.numeric(fit_ds$coefficients) -
                          as.numeric(fit_gauss$coefficients)))

  coef   = nm,
  dsVert = as.numeric(ds_b[nm]),
  lm     = as.numeric(rf_b[nm]))

At \(\lambda = 0\) the proximal-gradient handoff reproduces the prime federated Gaussian fit to within 0.00e+00. Against the centralised lm() reference the maximum absolute deviation is 7.02e-05, giving a σ-ratio of 675.3×.
